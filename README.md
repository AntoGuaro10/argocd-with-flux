# ArgoCD with FluxCD — GitOps Platform Repository

This repository is the core of my demo **GitOps** platform, structured using an hybrid architecture: **FluxCD** handles the underlying cluster infrastructure and component bootstrapping, while **ArgoCD** manages the entire lifecycle of application microservices.

The repository is built to integrate with a basic Internal Developer Portal (IDP), automating the creation and deployment of new microservices using a clean, multi-repo approach.

### Disclaimer: FluxCD "infrastructure" repository is obviously hidden 
---

## 🏗️ Architecture & Logical Flow

The flow strictly follows the GitOps philosophy and is designed to completely decouple infrastructure management from application definition:

```text
[ Developer Portal UI ] 
          │
          ▼
[ Portal Backend (Go) ] 
          │
          ├─► 1. Creates a new dedicated GitHub Repository for the microservice (e.g., 'payment-api')
          │      └─► Writes: k8s/app.yaml (Non-root Deployment + Service)
          │
          └─► 2. Registers the application in this central management repository
                 └─► Writes: apps/payment-api.yaml (ArgoCD Application)
                        │
                        ▼
              [ FluxCD Kustomization ] (Monitors the 'apps/' folder)
                        │
                        ▼ (Applies the resource onto the cluster)
              [ ArgoCD Application Controller ] 
                        │
                        ▼ (Points to the dedicated microservice repo)
              [ Application Pods in the Cluster ]
```

## FluxCD magic

Just create a `kustomization` object in your FluxCD controller and let ArgoCD deploy the Applications:
```yaml
apiVersion: kustomize.toolkit.fluxcd.io/v1
kind: Kustomization
metadata:
  name: argocd-apps
  namespace: flux-system
spec:
  interval: 1m0s
  path: ./apps
  prune: true
  sourceRef:
    kind: GitRepository
    name: flux-system
```
